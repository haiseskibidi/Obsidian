# Инфраструктурные модели (infrastructure/models.py)

**Суть в двух словах:** Чертежи таблиц для базы данных.

**Что делает:** 
Это стандартные модели Django ORM. Они описывают, какие колонки будут в таблицах SQLite/PostgreSQL, их длины, связи (`ForeignKey`, `OneToOneField`) и как они будут называться в админке (`class Meta`, `verbose_name`). По сути, это просто коробки для хранения информации (профили, логи аудита, история заявок).

**Чего тут НИКОГДА нет:** 
Любых сложных бизнес-правил, расчетов или проверок прав доступа. Эти классы нужны только для того, чтобы Django мог генерировать миграции и складывать данные на жесткий диск.
