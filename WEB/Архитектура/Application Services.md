# Сервисы (application/services.py)

**Суть в двух словах:** Дирижёр или главный менеджер процесса.

**Что делает:** 
Координирует всю работу (Сценарии использования/Use Cases). В сервисе прописан пошаговый алгоритм: 1. Достать пользователя из репозитория. 2. Достать заявку. 3. Спросить у доменной модели, имеет ли пользователь право её редактировать (и выкинуть `PermissionDeniedError`, если нет). 4. Обновить данные заявки. 5. Отдать команду репозиторию сохранить всё в базу.

**Чего тут НИКОГДА нет:** 
Никаких SQL-запросов напрямую в базу (это делает репозиторий) и никакого HTML или HTTP (это делают Views). Сервис занимается исключительно координацией бизнес-процесса.
