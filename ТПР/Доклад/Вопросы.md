# Вопросы к защите курсовой работы

1. **Что такое целочисленное линейное программирование (ЦЛП)?**
   Это раздел математического программирования, где переменные решения обязаны принимать целые значения. В моей задаче переменные бинарные — 0 или 1, то есть либо задача назначена на сервер, либо нет. Дробные решения здесь недопустимы — нельзя выполнить задачу «наполовину».

2. **Что такое целевая функция в вашей задаче?**
   Целевая функция — это то, что мы максимизируем. В моём случае это сумма приоритетов всех назначенных задач:
   $$Z = \sum_{i} \sum_{j} P_i \cdot x_{ij} \to \max$$
   Чем больше $Z$, тем более важные задачи мы выполнили.

3. **Почему задача NP-трудная?**
   Потому что это обобщённая задача о рюкзаке. У нас 24 бинарные переменные — это $2^{24} \approx 16,7$ миллионов комбинаций. Простой перебор невозможен для больших задач, и не существует алгоритма, который решал бы её за полиномиальное время. Поэтому нужен метод ветвей и границ, который отсекает бесперспективные ветви.

4. **Что такое метод ветвей и границ?**
   Это точный метод решения задач ЦЛП. Мы строим дерево решений: на каждой развилке фиксируем переменную в 0 или 1. Для каждой ветви считаем верхнюю оценку через ЛП-релаксацию. Если оценка ветви хуже уже найденного решения — ветвь отсекаем. Так мы обходим не всё дерево, а только перспективные части.

5. **Что такое ЛП-релаксация?**
   Это ослабление задачи: мы заменяем условие «переменная 0 или 1» на «переменная от 0 до 1». Задача становится непрерывной и решается быстро. Её оптимум — это верхняя граница (потолок) реального оптимума. Если даже потолок хуже нашего текущего рекорда — эту ветвь можно не рассматривать.

6. **Что такое инкумбент?**
   Инкумбент (рекорд) — это лучшее целочисленное решение, найденное на данный момент. Сначала он равен минус бесконечности (ничего не нашли). Когда находим первое допустимое целое решение — запоминаем его. Все дальнейшие ветви сравниваем с ним. Если потолок ветви $\le$ инкумбенту — отсекаем.

7. **Какие критерии отсечения существуют?**
   Три критерия:
   * **По оценке** — верхняя граница ветви не лучше текущего рекорда (инкумбента).
   * **По недопустимости** (по памяти) — ЛП-релаксация подзадачи не имеет допустимых решений (ресурсы не позволяют разместить задачи).
   * **По целочисленности** — нашли целое решение (оно автоматически становится новым рекордом, если его значение лучше текущего, дальше ветвиться не нужно).

8. **Откуда в ручном расчёте взялось число 0.875?**
   В ЛП-релаксации задача 3 (RAM = 1024 МБ) не помещается на S1 целиком, потому что часть ресурсов занята задачей 4. Свободно 896 МБ из 1024 нужных:
   $$896 \div 1024 = 0.875$$
   Это дробная доля — столько задачи «влезает» при ослабленном условии (релаксации).

9. **Что такое задача о рюкзаке?**
   Классическая задача: есть рюкзак грузоподъёмностью $W$ и набор предметов с весами и ценностями. Нужно выбрать подмножество предметов с максимальной ценностью, не превышая вес. В моей работе «рюкзаки» — это серверы, «предметы» — задачи, «ценность» — приоритет, «вес» — требования к RAM и VRAM.

10. **Чем GMMKP отличается от обычной задачи о рюкзаке?**
    Три усложнения:
    1. **Многомерность** — два ресурса: RAM и VRAM.
    2. **Множественность** — три сервера вместо одного рюкзака.
    3. **Обобщённость** — оптимизируем весь кластер серверов целиком.
    Из-за этих усложнений задача остаётся NP-трудной.

11. **Почему вы использовали Python и PuLP, а не другой язык?**
    Python — стандарт для научных расчётов с лаконичным синтаксисом. PuLP — удобная библиотека для декларативного описания моделей ЦЛП. Она генерирует модель и передаёт её решателю CBC. Это промышленный стек, который используется в реальных задачах оптимизации.

12. **Что такое CBC-решатель?**
    CBC (COIN-OR Branch and Cut) — это бесплатный высокопроизводительный решатель задач целочисленного программирования. Он реализует метод ветвей и границ с добавлением отсекающих плоскостей Гомори. Встроен в PuLP по умолчанию. Даёт гарантированно точное решение.

13. **Что означает статус «Optimal» в выводе программы?**
    Это означает, что решатель нашёл глобально оптимальное решение — лучше математически не существует. Все ветви дерева либо исследованы, либо корректно отсечены.

14. **Почему VRAM — узкое место, а не RAM?**
    Суммарная потребность задач в VRAM = 1920 МБ, а суммарная ёмкость серверов тоже = 1920 МБ. Это ровно 100% — ни одного мегабайта запаса. RAM же используется на 66–100% в зависимости от сервера. Именно VRAM ограничивает, сколько ещё задач можно добавить.

15. **Что такое бинарная переменная?**
    Это переменная, которая принимает только два значения: 0 или 1. В моей задаче $x_{ij} = 1$ означает «задача $i$ назначена на сервер $j$», а $x_{ij} = 0$ — «не назначена». Это моделирует решение типа «да/нет».

16. **Зачем нужен ручной расчёт, если есть программа?**
    Для верификации модели. Мы берём упрощённый пример (4 задачи, 2 сервера), считаем вручную методом ветвей и границ, получаем $Z^* = 23$ — и сравниваем с выводом программы. Результаты совпали — значит, математическая модель записана верно.

17. **Почему в ручном расчёте задача 4 не была назначена?**
    Потому что оба сервера заполнены более приоритетными задачами. Задача 3 (приоритет 9) заняла весь S1, задачи 1 и 2 (приоритеты 8 и 6) заняли весь S2. Задача 4 (приоритет 4) — наименее важная из оставшихся, для неё физически нет места. Выбрасывается наименее ценная — это оптимально.

18. **Можно ли было использовать симплекс-метод?**
    Симплекс-метод решает непрерывные задачи линейного программирования (ЛП), но не гарантирует целочисленность. Если просто округлить его ответ, можно получить недопустимое или неоптимальное решение. Поэтому для задач с бинарными переменными нужен именно метод ветвей и границ, а симплекс используется внутри него для решения ЛП-релаксаций на каждом узле дерева.

19. **Что такое ограничение уникальности назначения?**
    Это условие: каждая задача может быть назначена максимум на один сервер. Формально для каждой задачи $i$:
    $$\sum_{j} x_{ij} \le 1$$
    Нельзя, чтобы одна задача одновременно выполнялась на двух серверах. Также допускается, что задача не будет назначена вовсе (сумма = 0).

20. **Какова практическая значимость вашей работы?**
    Разработанная модель и программа могут быть интегрированы в балансировщики нагрузки облачных платформ и рендеринг-ферм. Это позволит автоматически распределять задачи по серверам, снижая простои оборудования и обеспечивая приоритетную обработку критически важных задач.
